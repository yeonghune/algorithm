# BOJ 1202 - 보석 도둑

- 링크: https://www.acmicpc.net/problem/1202
- 난이도: 골드 2
- 분류: 그리디, 우선순위 큐
- 정리 일자: 2025-11-30

## 문제 설명

`N`개의 보석이 있고 각 보석은 무게 `M`, 가격 `V`를 가진다. `K`개의 가방은 각각 무게 제한 `C`를 갖는데, 가방 하나에는 보석 하나만 넣을 수 있다. 모든 가방에 들어간 보석 가격의 합을 최대로 만드는 것이 목표다.

## 입력

1. 첫 줄에 보석 개수 `N (1 ≤ N ≤ 300,000)`과 가방 개수 `K (1 ≤ K ≤ 300,000)`이 주어진다.  
2. 다음 `N`줄에 각 보석의 무게 `M`과 가격 `V`가 주어진다. (`0 < M, V ≤ 1,000,000`)  
3. 이후 `K`줄에 각 가방의 무게 제한 `C`가 주어진다. (`0 < C ≤ 100,000,000`)

## 출력

가방에 담을 수 있는 보석 가격 합의 최댓값을 출력한다.

## 예시

| 입력 | 출력 |
|------|------|
| `3 2`<br>`1 65`<br>`5 23`<br>`2 99`<br>`10`<br>`2` | `164` |

## 메모

- 가방을 무게 오름차순으로 훑으면서, 현재 가방에 들어갈 수 있는 보석을 모두 최대 힙(가격 기준)에 넣고 가장 비싼 보석을 꺼내면 된다.  
- 각 보석은 최대 한 번 push/pop 되므로 전체 복잡도는 `O((N+K) log N)`이다.
